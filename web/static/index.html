        About DICOM Structured Reports
      
      
      
      

        
      
          
      What is a Structured Report?
      
          
      A DICOM Structured Report (SR) encodes clinical information in a machine-readable, hierarchical tree of coded concepts. Unlike a PDF scan or plain text, SR content can be queried, extracted, and validated by software — enabling dose monitoring, CAD integration, measurement tracking, and decision support workflows.
      
          
      The document title (shown in the meta bar after parsing) comes from the root (0040,A043) ConceptNameCodeSequence and identifies what kind of SR you are looking at — for example Diagnostic imaging report (LN:81220-6) for a radiology report, or X-Ray Radiation Dose Report (DCM:113701) for a dose SR.

            ORM field mapping (advanced)
            
      Map HL7 segment fields (e.g. OBR.3) to DICOM metadata. Changes are saved to configuration.
      
            
      
              When an ORM/ORU message is received, PACSAdminTool extracts patient and study data
              from the HL7 segments listed here. Each key is a DICOM field name and each value
              is an HL7 segment reference in SEGMENT.field[.component]
              notation (e.g. "PatientName": "PID.5.1").
              Unmapped fields fall back to built-in defaults. Expand the JSON editor below,
              adjust the mapping to match your HL7 sender, then click Save mapping.
